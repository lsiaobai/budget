廣告預算核對雙簽表格 (C-BVC)這是一個基於 HTML、Tailwind CSS 和 Firebase Firestore 構建的單頁 Web 應用程式，旨在提供一個標準化的流程來核對廣告活動的預算設定，以避免人為錯誤和預算超支。核心功能：雙簽核對: 提供「執行設定」和「檢核設定」兩組欄位，確保重要數據經過兩人核對。雲端共享歷史記錄: 所有記錄都儲存在 Firebase Firestore 的公共路徑下，確保團隊成員之間可以在不同設備上即時共享和查看歷史資料。條件式必填: 根據選擇的「檢核類型」（帳戶預算、廣告預算或全部），動態設定表格中的必填欄位。數據導出: 支援將所有歷史記錄導出為 CSV 格式。用戶識別: 在共享環境中，會顯示創建該筆記錄的使用者 ID。部署與運行檔案結構: 專案僅包含兩個檔案：index.html: 包含所有 HTML、CSS (Tailwind) 和 JavaScript (包括 Firebase 邏輯)。README.md: 本說明文件。Firebase 設置:此應用程式依賴 Firebase Firestore 進行數據持久化。由於運行環境自動提供了 Firebase 配置 (__firebase_config 和 __app_id) 和認證令牌 (__initial_auth_token)，您無需手動配置 Firebase 金鑰。數據儲存路徑為：/artifacts/{appId}/public/data/budget_checklists。上傳至 GitHub:建立一個新的 Git 儲存庫。將 index.html 和 README.md 上傳到儲存庫的根目錄。部署: 您可以將此 HTML 檔案部署到任何靜態網站服務（例如 GitHub Pages, Netlify, Vercel）運行。數據共享說明 (重要)共享機制: 由於程式碼將數據儲存在 Firebase 的 公共路徑，任何擁有此應用程式存取權限的人都將共享相同的歷史紀錄清單。用戶識別: 每個記錄都會自動帶上 createdByUserId 欄位，以追蹤是哪位用戶創建或修改了這筆數據。
